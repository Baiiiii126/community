# RedAI Community

欢迎来到 RedAI Community！这里是小红书 AI Infra 团队面向开源社区的协作入口，用于沉淀社区活动、贡献指南和协作资料。具体项目的代码开发、Issue 与 Pull Request 请前往对应项目仓库。

我们致力于建设领先、开发者友好的 AI 大模型全栈基础设施。无论你是第一次参与开源，还是希望深入大模型训练、推理、优化与服务，都欢迎从一个 Issue 开始。

## 社区活动

### Relax 进校园 · 第一期开放任务

该活动面向希望参与大模型强化学习与 AI Infra 开源建设的开发者，提供新手入门任务和正式贡献任务。你可以先阅读活动说明，再根据自己的兴趣与经验选择任务。

- [活动流程与参与说明](https://github.com/redai-infra/Relax/issues/77)
- [新手入门任务](https://github.com/redai-infra/Relax/issues/76)
- [浏览全部开放任务](https://github.com/redai-infra/Relax/issues)

活动安排与任务状态以对应 Issue 中的最新信息为准。

## 如何参与

你可以通过以下方式参与 RedAI 开源社区：

- 报告问题，提出功能建议或分享使用反馈；
- 认领 Issue，修复缺陷、补充测试或优化性能；
- 完善文档、教程、示例和开发者工具；
- 参与代码评审与技术讨论；
- 分享实践经验，参与社区活动。

### 贡献流程

1. 在对应项目中查找或创建 Issue，说明问题背景、目标和预期结果。
2. 在 Issue 下留言认领任务，并与维护者确认实现范围和验收标准。
3. Fork 项目并创建独立分支，完成代码、测试和必要的文档更新。
4. 提交 Pull Request，在描述中关联 Issue，并说明改动内容和验证结果。
5. 根据评审意见继续完善，直至改动合入。

提交贡献前，请阅读项目的[贡献指南](https://github.com/redai-infra/Relax/blob/main/docs/zh/guide/how-to-contribute.md)。

## 社区协作原则

- **开放透明**：重要讨论和决策尽量在公开的 Issue、Discussion 或 Pull Request 中进行。
- **尊重友善**：尊重不同背景与观点，围绕事实和技术问题展开讨论。
- **任务清晰**：Issue 应尽量写明背景、目标、范围和验收方式。
- **结果可复现**：代码贡献应提供必要的测试、日志、环境说明或实验结果。
- **安全合规**：发现安全问题时，请按照项目的[安全政策](https://github.com/redai-infra/Relax/blob/main/.github/SECURITY.md)进行反馈。
